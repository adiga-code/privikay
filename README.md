# Habit Tracker Bot — @Privykai_bot

Telegram-бот для формирования ежедневных привычек здоровья. Пользователь тратит 20–60 секунд в день на чек-ин, получает статистику, стрики и мотивацию от выбранного героя. Монетизация — подписка через ЮКасса.

---

## Возможности

- **Онбординг** — 8 шагов: имя, контакт, цель, цель по весу, выбор привычек, часовой пояс, настройка целей (шаги/калории/время чек-ина/время сна), выбор героя
- **Ежедневный чек-ин** — напоминание в заданное время, вопросы по выбранным привычкам, Индекс дня (0–10)
- **8 привычек** — шаги, питание, сон, стресс, энергия, алкоголь, курение, без сахара
- **Недельный отчёт** — каждые 7 дней: статистика, средний индекс, стрики, карточка прогресса с кнопкой «Поделиться»
- **Контроль веса** — еженедельное взвешивание с динамикой
- **8 героев** — Капибара, Ленивец, Блоб, Хомяк, Аксолотль, Росток, Дракон, Робот — каждый со своим характером
- **Напоминание о сне** — мягкое сообщение за 30 минут до заданного времени сна
- **Подписка (ЮКасса)** — 10 дней бесплатно → выбор плана → ссылка на оплату → автоактивация
- **Админ-панель** — статистика пользователей, список активных подписок

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone <repo-url>
cd habit_bot
```

### 2. Создать виртуальное окружение

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# Linux / macOS
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Настроить переменные окружения

Скопировать `.env.example` в `.env` и заполнить:

```bash
cp .env.example .env
```

### 5. Запустить

```bash
python bot.py
```

При первом запуске база данных создаётся автоматически.

## Конфигурация (.env)

```env
# Токен бота от @BotFather
BOT_TOKEN=1234567890:AAXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

# Имя бота (для return_url после оплаты)
BOT_USERNAME=YourBotName

# URL базы данных (SQLite по умолчанию)
DB_URL=sqlite+aiosqlite:///./habits.db

# URL академии (кнопка в оффере после 7 дней)
ACADEMY_URL=https://example.com/academy

# ЮКасса — формат: shopId:MODE:secretKey
# MODE = TEST или LIVE
YUKASSA_TOKEN=1023167:LIVE:live_xxxxxxxxxxxxxxxxxxxxxxxxxx

# Telegram ID администраторов через запятую
ADMIN_IDS=123456789,987654321
```

### Получение токена ЮКасса

1. Зайти в [ЮКасса → Настройки магазина](https://yookassa.ru/my/shop-settings/payments)
2. Скопировать **shopId** и **Секретный ключ** (live-режим)
3. Вставить в `.env` в формате `shopId:LIVE:live_secretKey`

> **Важно:** В аккаунте ЮКасса должна быть включена возможность создания платёжных ссылок. Если включена отправка чеков (54-ФЗ), бот автоматически запрашивает email или использует телефон пользователя для формирования чека.

| | Цена |
|---|---|
| Бесплатный период | 10 дней |
| Месяц | 249 ₽ |
| Год | 1 790 ₽ (экономия 1 198 ₽) |

**Сценарий оплаты:**
1. `/subscribe` → выбрать план
2. Бот создаёт платёж в ЮКасса → отправляет ссылку
3. Пользователь оплачивает на странице ЮКасса
4. Нажимает «✅ Я оплатил» → бот проверяет статус и активирует подписку

## Фоновые задачи (APScheduler)

| Задача | Расписание |
|---|---|
| Напоминания о чек-ине | Каждую минуту (сверяет с `checkin_time` пользователей) |
| Напоминание о сне | Каждую минуту (за 30 мин до `sleep_target_time`) |
| Недельный отчёт | Ежедневно в 00:05 UTC (раз в 7 дней на пользователя) |
| Напоминание взвешивания | Ежедневно в 00:05 UTC (раз в 7 дней) |
| Инсайты по данным | Ежедневно в 00:05 UTC (раз в 4 дня) |
| Оффер академии | Ежедневно в 00:05 UTC (однократно, после 7 дней) |

## Архитектурные решения

- **Strategy Pattern** — каждая привычка изолирована в классе (`validate`, `evaluate`, `question`). Добавить новую = создать класс + одна строка в `registry.py`
- **Динамические FSM-очереди** — онбординг и чек-ин используют `queue: list[str]` в FSM data вместо жёстких состояний под каждый вопрос
- **Middleware-цепочка** — `DatabaseMiddleware` → `SubscriptionMiddleware`. Порядок важен и зафиксирован в `bot.py`
- **Чистые сервисы** — `AnalyticsService` и `SubscriptionService` не обращаются к БД, тестируются без неё
- **Единый источник правды** — фразы героев в `heroes/data.py`, привычки в `habits/registry.py`, настройки в `.env`
